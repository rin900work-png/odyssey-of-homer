# CLAUDE.md

Инструкции для Claude Code при работе с этим репозиторием.

Это **самостоятельный проект**, не связанный с другими репозиториями (в т.ч. с `wb_sales`).
Общего кода, зависимостей и общей памяти нет.

## Что это

Маленький **статический сайт** — краткий разбор поэмы Гомера «Одиссея» по всем 24 песням.
Стек намеренно простой: **чистый Python 3 (стандартная библиотека), без Flask, без сервера,
без `pip install`**. Генератор собирает HTML, который раздаётся как статика.

- Живой сайт: https://rin900work-png.github.io/odyssey-of-homer/
- Remote: `github.com/rin900work-png/odyssey-of-homer` (ветка `main`)

## Команды

```bash
# Пересобрать сайт (docs/)
python3 build.py

# Проверка синтаксиса перед коммитом
python3 -m py_compile build.py content.py

# Посмотреть локально — просто открыть docs/index.html в браузере
open docs/index.html

# Опубликовать: GitHub Pages раздаёт docs/ из ветки main
python3 build.py && git add -A && git commit -m "..." && git push
# Сайт обновится за ~минуту
```

## Архитектура

Два файла с логикой + сгенерированный вывод:

- **`content.py`** — ВСЕ данные (чистый Python, без зависимостей): `META`, `PARTS` (3 части),
  `CHARACTERS`, `THEMES`, `SONGS` (24 песни), плюс привязка картинок `_SONG_IMG` / `_CHAR_IMG`
  в конце файла. Содержание правится ЗДЕСЬ.
- **`build.py`** — генератор: читает `content.py` → собирает страницы в `docs/`. Экранирование
  HTML через `html.escape` (обёртка `e()`), поэтому в тексты `content.py` кладём обычный текст,
  а не сырой HTML.
- **`static/style.css`** — стили (тема «эгейское море»: пергамент + синь).
- **`static/img/`** — картинки-исходники (общественное достояние, Wikimedia Commons).
- **`docs/`** — СГЕНЕРИРОВАННЫЙ сайт: `index.html`, `song-01..24.html`, `style.css`, `img/`,
  `.nojekyll`.

**Композиция (3 части):** I. Телемахия (песни 1–4) · II. Странствия (5–12) ·
III. Возвращение и месть (13–24).

**Иллюстрации:** к каждой песне (кроме 8 и 15) и к каждому персонажу подобрана картина/гравюра
в общественном достоянии; под каждой — подпись `credit` с автором и названием.

## Критические правила

- **`docs/` руками не редактировать.** При каждой сборке `build.py` делает `shutil.rmtree(docs/)`
  и пересоздаёт папку — ручные правки затрутся. Меняем `content.py` / `static/`, потом
  `python3 build.py`.
- **Всегда пересобирать перед коммитом**, чтобы `docs/` соответствовал исходникам, иначе живой
  сайт разойдётся с `content.py`. В один коммит идут и исходники, и пересобранный `docs/`.
- **Никаких внешних зависимостей** — только стандартная библиотека Python 3. Проект должен
  собираться командой `python3 build.py` на голом Python.
- **`.nojekyll`** нужен, чтобы GitHub Pages раздавал файлы как есть (без обработки Jekyll);
  его пишет `build.py` — не удалять.
- Новые картинки кладём в `static/img/` (не в `docs/img/`) и привязываем в `content.py`;
  `build.py` сам скопирует их в `docs/img/`. Использовать только public domain / свободные
  лицензии, с подписью-`credit`.
- `.DS_Store`, `__pycache__/`, `*.pyc` — в `.gitignore`, не коммитить.

> Примечание: папка `songs/` пустая — исторический артефакт, содержимого там нет.

## Система памяти проекта

В корне проекта — папка `brain/` (в репозитории):

- `brain/daily.md` — журнал сессий. Читать в начале сессии; **дозаписывать** (append) после
  заметных задач и перед паузой/`/compact`, не только в конце.
- `brain/decisions.md` — архитектурные решения: почему так, какие альтернативы отвергли.
- `brain/lessons.md` — баги и паттерны: сложные баги и как решили, что работает/не работает.

Формат записи в `daily.md`:

```
### [YYYY-MM-DD]
- Что сделали
- Что не получилось и почему
- Что делать дальше
```

Память Claude у проекта своя, отдельная от других репозиториев (в т.ч. `wb_sales`).
